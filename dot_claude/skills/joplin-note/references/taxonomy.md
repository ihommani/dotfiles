# Joplin Tag Taxonomy

Pick **2–5 tags** per note. Never invent tags outside this list.

## Tech / Dev
`development` · `infrastructure` · `container` · `security` · `networking` · `database` · `cloud` · `ai` · `automation` · `frontend` · `backend`

## Personal
`cooking` · `food` · `fitness` · `health` · `finance` · `home` · `administrative`

## Leisure
`trip` · `outing` · `entertainment` · `reading` · `media`

> `media` = entertainment you consume for leisure (films, TV shows, podcasts as hobby). Do **not** use it for notes whose *content* is technical even if the source was a podcast or video.

## Work / Meta
`meeting` · `project` · `reference` · `research` · `idea` · `todo` · `writing` · `learning` · `pro`

---

## Selection rules

- Prefer the **broadest accurate category** that still narrows the search meaningfully.
- Avoid stacking tags that mean the same thing at different granularities (e.g., don't use both `trip` and `outing` for a weekend day-trip).
- Fewer is better. Two good tags beat five noisy ones.

### Easily confused tags

| Tag | Use when… |
|---|---|
| `idea` | Recording your own original thought or brainstorm |
| `research` | Ongoing investigation or exploration of a topic |
| `learning` | Capturing knowledge absorbed from an external source (book, course, article) |
| `reference` | Compact, distilled material meant to be looked up again later |

## Examples

| Content | Tags |
|---|---|
| Notes from a team standup | `meeting` · `project` |
| How to set up a Redis container | `container` · `database` · `development` |
| Weekend trip to Barcelona | `trip` |
| Recipe for beef stew | `cooking` |
| Book summary: The Pragmatic Programmer | `reading` · `reference` |
| Home renovation budget | `home` · `finance` |
| Ideas for a new side project | `idea` · `development` |
| Restaurant review | `food` · `outing` |
| Online course notes | `learning` · `pro` |
| Tax return paperwork | `administrative` · `finance` |
